# 宿舍保修系统 - 第二周

## 一、项目简介
本项目为宿舍报修管理系统的后端与前端实现，基于SpringBoot3+MyBatis-Plus+JWT提供RESTful API，前端使用Vue3+Vite+axios构建用户界面。实现了学生注册登录、宿舍绑定、报修单创建（含图片上传）、报修单取消与查询，以及管理员对报修单的增删改查等完整功能。

## 二、技术栈
### -后端
- Java 17
- SpringBoot 3
- MyBatis-Plus
- JWT
- MySQL
### -前端
- Vue3
- Vite
- axios
- HTML/CSS

## 三、功能实现

## 四、JWT登录鉴权实现与存储说明
### 实现方式
#### 1.Token生成
用户登录成功后，后端使用JWTUtil工具类生成JWT，其中包含由引入的JJWT库自动生成的header；userId和role作为payload，并设置过期时间为24小时；最后的signature使用HS256算法，密钥配置在application.yaml中。
#### 2.Token校验
自定义AuthInterceptor拦截器，拦截所有HTTP请求（除登录、注册、静态资源外），从请求头Authorization中提取Token，验证签名和有效期。校验通过后将userId和role存入请求属性，供后续controller使用。失败则返回401。
#### 3.接口放行
登录接口、注册接口、静态资源路径配置和预检请求为放行，不进行Token校验。
### 存储方式
#### 后端
不保存Token，仅在生成时返回给前端，后续请求由前端携带Token进行鉴权。
#### 前端
登录成功后将Token存储在localStorage中。后续请求通过axios拦截器自动从localStorage读取token并添加到请求头中。

## 六、遇到的问题与解决方案
1. pom.xml第一行注释导致解析错误
    - 将xml声明移到第一行
2. 登录接口返回401
   - 修改拦截器，放行登录接口
3. 图片上传后访问路径错误 
   - 修改后端图片访问路径为完整的URL
4. 图片访问返回304
   - 配置静态资源映射，确保路径正确
5. 前端端口自动递增
   - 使用npm run dev -- --port 5173固定端口
6. 取消保修单返回404
    - 修改接口路径，确保前端请求路径与后端一致
7. 状态数字显示不友好
    - 在前端添加状态映射，将数字转换为文字显示
8. 学生首页显示所有保修单
    - 修改接口，学生只能查询自己的保修单列表
9. 学生首页未绑定时显示创建表单
   - 增加bound状态判断，未绑定时仅显示绑定表单
10. 取消保修单后状态未更新
    - 修改接口逻辑，取消后将状态更新为已取消
11. 添加上按优先级筛选后，无法同时按状态和优先级筛选
    - 修改接口逻辑，支持同时按状态和优先级筛选
12. 添加上同时按状态和优先级筛选后，使用的时候每次都是只按优先级查询
